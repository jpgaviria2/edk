## Purpose

The imported curriculum is the educational source layer for building EDK practical games, activities, and interactive lessons. The EDK block folders are the product/design layer.

The platform direction is now **12 practical Bitcoin learning blocks**. Each block should eventually include:

- **Learn** — concise curriculum-based explanation
- **Try** — interactive practical example/game
- **Discuss** — facilitator prompts
- **Apply** — real-world scenario
- **Build** — printable/classroom activity

## 12-block platform roadmap

| EDK Block | Main curriculum sources | Practical build direction |
|---|---|---|
| Block 1 — History of Money | `1-what-is-money`, `2-the-history-of-money`, `3-what-is-fiat-money`, `4-how-problems-lead-to-solutions` | Barter, money functions/properties, inflation, debasement, Bitcoin bridge |
| Block 2 — What Is Money? | `1-what-is-money` | Definition of money, money psychology, types of money, value measurement games |
| Block 3 — Fiat, Inflation & Purchasing Power | `3-what-is-fiat-money`, `4-how-problems-lead-to-solutions` | Fiat systems, money printing, debt, prices, wages, purchasing-power simulators |
| Block 4 — Why Bitcoin? | `4-how-problems-lead-to-solutions`, `5-what-is-bitcoin` | Match fiat/trust problems to Bitcoin design choices |
| Block 5 — Wallets, Keys & Custody | `6-how-to-use-bitcoin`, `11-back-matter` | Wallet types, seed phrases, self-custody, recovery habits, safety scenarios |
| Block 6 — Transactions: Send & Receive | `6-how-to-use-bitcoin`, `7-using-bitcoin-in-daily-life` | Addresses, invoices, QR flow, transaction review, don't-trust-verify habits |
| Block 7 — Cold Storage | `6-how-to-use-bitcoin`, `10-what-future-can-bitcoin-build`, back matter | Cold storage, backups, threat modeling, recovery planning |
| Block 8 — Lightning & Daily Payments | `7-using-bitcoin-in-daily-life` | Lightning invoices, routing, wallet tradeoffs, merchant/coffee payment simulator |
| Block 9 — Proof of Work & Mining | `8-how-bitcoin-works`, `9-how-does-bitcoin-mining-work` | Hashing, nonces, difficulty, mining races, block creation, mempool inclusion |
| Block 10 — Nodes, Rules & Consensus | `4-how-problems-lead-to-solutions`, `5-what-is-bitcoin`, `9-how-does-bitcoin-mining-work` | Node validation, invalid block rejection, decentralized rules, Byzantine coordination |
| Block 11 — UTXOs, Fees & Privacy | `8-how-bitcoin-works/2-the-utxo-model`, `9-how-does-bitcoin-mining-work` | UTXO puzzles, coin selection, fee sizing, change outputs, privacy leaks |
| Block 12 — Bitcoin Future & Philosophy | `10-what-future-can-bitcoin-build`, `11-back-matter/1-real-world-case-studies.md` | CBDCs, censorship resistance, financial empowerment, sovereignty, real-world cases |

Each block folder has or should have a `CURRICULUM.md` file with detailed source links and practical game opportunities.

## Legacy/current folder note

The original EDK repo already had seven block folders. Some historical folder names still reflect the first workshop sequence. Going forward, the 12-block roadmap above is the product roadmap; older folders can be renamed or migrated as each block gets built.
